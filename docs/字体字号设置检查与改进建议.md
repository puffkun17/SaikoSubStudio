# 字体字号设置检查与用户体验 / 审美改进建议

**日期**：2026-06  
**项目**：SaikoSubStudio（影院级双语字幕预览与编辑工具）  
**检查范围**：预览渲染（ScreenSimulator）、样式控制（StyleSidebar + ControlDeck）、状态模型（useStudioStore + StyleSettings）、导出/导入（subtitleCore）、全局 UI 字体。

---

## 1. 当前实现概览（字体与字号）

### 1.1 预览渲染核心（ScreenSimulator.tsx）
- 使用 **容器查询高度单位 (cqh)** 实现响应式字号：
  ```ts
  const zhSizeCqh = (style.zhFontSize * scale / 288) * 100;
  const enSizeCqh = (style.enFontSize * scale / 288) * 100;
  // 同理用于 lyric、note、marginV
  ```
- **魔法数字 288**：与 ASS 导出中的 `resY / 288` multiplier 保持一致（历史设计，模拟旧版 ASS 参考分辨率）。
- 字体硬编码：
  - 中文/注释：`system-ui, sans-serif` + fontWeight 700/500
  - 英文/歌词英：`Helvetica Neue, Arial, sans-serif` + 部分 enScale transform
- 描边：8 层 text-shadow（无真实 stroke）。
- 歌词独立处理（字号、颜色、斜体、top/bottom 位置）。
- globalScale 存在于模型，但**UI 中未暴露**。

### 1.2 样式模型（StyleSettings + store）
```ts
{
  zhFontSize: 20, enFontSize: 12,
  lyricFontSize: 16,
  marginV: 20,
  globalScale: 1.0,
  // + colors, enScale, resolution, aspectRatio...
}
```
- 默认值合理，但“px”标签误导用户。
- 三个预设（Netflix / 大银幕 / 动漫）在 ControlDeck 中定义，效果不错但数量少。

### 1.3 UI 控制（StyleSidebar.tsx）
- 滑块范围：ZH 12-36、EN 8-24、歌词 10-30、Margin 10-60。
- 有“参数微调”分区 + 歌词折叠区。
- 标签仍写 “XX px”，实际是参考单位。
- 缺少：
  - globalScale 控件
  - 字体家族选择
  - 精确数字输入
  - 字号与真实 ASS 输出的对应说明

### 1.4 导出/导入（subtitleCore.ts）
- 导出时用 `multiplier = resY / 288 * globalScale` 换算真实 ASS Fontsize。
- 导入 ASS 时用 `/ 3.75` 的经验公式把 Fontsize 转回参考单位（脆弱）。
- 硬编码字体名：`PingFang SC`（中）、`Helvetica Neue`（英）。

### 1.5 全局 UI 字体
- 使用 Geist Sans / Geist Mono（现代、优秀的中英混排支持）。
- 预览画布内的字幕字体与 App UI 字体分离（正确）。

## 2. 审美需求分析（影院级字幕预览工具）

目标用户：字幕工作者、影视爱好者、翻译者。核心场景是**所见即所得的影院模拟**。

**审美要求**：
- 电影感（filmic）：干净、优雅、有呼吸感，不要太“软件味”。
- 中英混排和谐：中文通常需要稍重或特定 CJK 字体；英文需要良好字重与间距。
- 在不同背景（暗夜、自然光、电影院、TMDB 剧照）下都高可读性。
- 不同画幅（4:3 CRT、宽银幕、IMAX）下字号感知一致。
- 导出 ASS 后在真实播放器（VLC / mpv / PotPlayer）里的效果要接近预览。

**用户体验痛点**（当前版本）：
- “字号 xx px” 让人困惑（实际不是屏幕像素）。
- 无法选择字体 → 不同操作系统预览差异大（尤其是中文）。
- 缺少“整体放大/缩小”一键控制（globalScale 藏着）。
- 滑块无精确输入，微调困难。
- 描边用纯 shadow，在小字号或高分辨率下不够锐利。
- 预设少，用户经常要手动调很多参数。
- 导入 ASS 后字号经常“跑偏”，需要反复调整。

## 3. 具体改进意见（按优先级）

### 高优先级（强烈推荐立即实现）

**1. 暴露 globalScale + 改进标签体系**
- 在“参数微调”区增加“整体缩放 (Global Scale)”滑块（0.6 ~ 1.8，步进 0.05）。
- 把所有字号标签从 “中文字号 (ZH Font Size) 20px” 改成：
  - “中文字号 (参考单位 / ASS)”
  - 右侧同时显示 “预览 ≈ XX cqh” 或 “相当于 1080p 下约 XX pt”。
- 添加小提示或 tooltip：“这些数值会按分辨率与 globalScale 换算到最终 ASS 字号”。

**2. 增加字体家族选择（最重要审美提升）**
- 在 StyleSidebar 增加“字体”分区：
  - 中文字体族：系统默认 / 苹方 (PingFang SC) / 黑体 / 思源黑体 (Noto Sans SC) / 自定义
  - 英文字体族：系统默认 / Helvetica Neue / Arial / Inter / 电影常用无衬线
- 在 ScreenSimulator 中把 `fontFamily` 应用到对应元素。
- 在 generateAssContent 中使用用户选择的 Fontname（回落到当前硬编码）。
- 这能让用户做出真正“专业”的预览效果，不同设备也更一致。

**3. 滑块 + 数字输入双控件**
- 每个字号/ margin 滑块旁边放一个小 number input（受控，同步更新 store）。
- 极大提升专业用户微调效率。

**4. 优化描边渲染（视觉质量）**
当前 8 层 shadow 太重且偏软。建议：
```css
textShadow: getOutlineShadow(...),
WebkitTextStroke: `${outlineWidth}px ${outlineColor}`,
paintOrder: 'stroke fill',
```
并减少 shadow 层数或强度。现代浏览器支持很好，预览会更锐利、专业。

### 中优先级

**5. 丰富并优化预设**
- 当前 3 个不错，建议增加：
  - “现代简洁 (Netflix 202x)”
  - “胶片经典 (Film Grain)”
  - “IMAX / 大银幕沉浸”
- 让预设同时影响 globalScale、lyricItalic、enScale 等更多参数。
- 在预设按钮上显示小预览缩略（或 hover 描述“推荐用于 2.39:1 宽银幕”）。

**6. 改进 ASS 导入字号映射**
`/3.75` 是经验值，容易在不同 resY 下出问题。
建议：
- 记录 ASS 原始 PlayResY + 原始 Fontsize。
- 提供“按原始 ASS 参考分辨率还原”按钮。
- 或者让用户在导入后看到“检测到原始字号 XX，是否按比例映射？”

**7. 增加更多微调项（进阶但有价值）**
- 字间距 (Letter Spacing)
- 行高 (Line Height)
- 英文字重独立控制
- 描边粗细独立滑块（当前硬编码在导出里）

### 低优先级 / 长期

- 支持 web font 加载（用户上传字体或从 Google Fonts / 字库选）。
- 提供“匹配当前视频分辨率”一键按钮（结合用户上传的视频元数据）。
- 在预览右下角实时显示“当前设置对应的 ASS PlayResY + 字号”。
- 暗黑模式下字号对比度自动建议（基于当前 backdrop 亮度）。

## 4. 实施建议与风险

- **向后兼容**：globalScale 默认 1.0，所有老存档/模板都能正常加载。
- **性能**：cqh + container query 已经很好，加 text-stroke 几乎无成本。
- **导出一致性**：改动 fontFamily 后必须同步修改 generateAssContent，否则“预览漂亮、导出丑”的问题会加剧。
- **测试重点**：
  - 不同画幅下字号感知是否一致。
  - 中英混排在各种背景（尤其是新生成的 scene_nature / night）下的可读性。
  - ASS 导出后在 VLC / mpv 里打开的效果接近度。
  - localStorage 持久化是否正确（尤其是新增的 fontFamily）。

---

**结论**：

当前字体字号系统**基础扎实**（cqh 响应式 + 288 参考单位 + 歌词独立处理），但在“专业感”和“易用性”上还有明显提升空间。**添加字体选择 + 暴露 globalScale + 清理“px”误导标签** 这三件事就能让工具的审美档次和用户信心显著上升。

建议优先实现高优先级 1-4 项，配合之前已经完成的“预览场景与剧照机制”文档，逐步把 SaikoSubStudio 打造成真正“所见即所得”的影院级字幕工作台。
